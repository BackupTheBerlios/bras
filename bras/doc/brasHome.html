Although it is one of the best tools around, sometimes I find working with make a bit annoying, and I assume everbody knows his or her most disliked misfeatures of make, so I will not repeat them here.

If you know the talking, you can immediately download bras.tar.gz (~100k) , read the online documentation or download the documentation alone.

Otherwise you may want to have a look at the main features first.

REMARK: Some of the features mentioned below are not yet available in the version you can download. They are however already implemented. I am testing the new release and updating the docs.
  • rule-controlled command execution, similar to make;
  • totally written in Tcl, no compilation required;
  • Tcl-syntax, so if you like Tcl, you will like bras;
  • dependencies in foreign directories are handled nicely;
    If a dependency starts with @ and contains a pathname, the rule file of this directory is automatically read, assuming that it contains the rules for making the dependency. In particular, no recursive call of bras is necessary, i.e. the chain of reasoning is not broken.
  • more than one type of rule;
    A reason for making a target is usually that it is older than its dependencies. However bras has several types of rules implementing other reasons. The Exist-rule fires only, if the target does not exist. The Always-rule fires always.
  • new types of rules can be easily added;
    If the standard-types of rules are not enough, bras allows to define new types of rules. Maybe you need a rule that makes its target if it is smaller than a dependency.
  • clear distinction between dependencies and mere prerequisites;
    Sometimes the command to update a target needs additional files beside the dependencies which have otherwise nothing to do with the target. In particular, they should not cause the rule to fire. Those files can be clearly differentiated from the dependencies.
  • multiple targets in one rule;
    This is typically necessary, if a command creates more than one file.
  • multiple rules for a target;
    If one rule fails because a dependency cannot be made, maybe the next one works.
  • pattern rules which deserve the name;
    The target of a pattern rule is a regular expression. To derive the dependency in case the target matches, any Tcl-proc can be called.
Bras is new and therefore not yet fully developed. Features which are currently missing are:
  • parallel execution of commands (like gnu make's -j option);
  • automatic conversion of makefiles;
  • private variable scope for rule-files in different directories;
  • application of pattern rules in a way which allows to bridge several missing links;
